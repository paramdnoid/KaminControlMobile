# Mobile Design System Reference

## Existing Tokens

- Background: `colors.background`, surfaces: `colors.surface`, `colors.surfaceMuted`.
- Text: `colors.text`, muted text: `colors.muted`.
- Primary action: `colors.primary`, `colors.primaryPressed`, `colors.primarySoft`.
- Accent/status: `colors.accent`, `colors.success`, `colors.warning`, `colors.danger`, `colors.info`.
- Spacing: `xs`, `sm`, `md`, `lg`, `xl`, `xxl`.
- Radius: `sm`, `md`, `lg`.
- Type: `title`, `h2`, `h3`, `body`, `small`, `label`.

## UI Principles

- Field users need fast scanning and reliable touch interactions more than decoration.
- Keep primary actions obvious and available near the workflow they complete.
- Use cards for grouped information and repeated items, not for every page section.
- Prefer clear labels, stable spacing, and predictable stacking on narrow screens.
- Avoid one-note palettes, excessive shadows, oversized hero patterns, and decorative UI that hides operational data.

## Expected States

- Import: empty, picking, preview, warning, saving, saved.
- Search/home: no data, loading, results, creating report.
- Property detail: not found, Genesis context present/missing, PDF unavailable.
- Report wizard: autosave, validation errors, suggestion applied, completed/exported.
